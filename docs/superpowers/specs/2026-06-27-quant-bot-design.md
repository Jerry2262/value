## 1. 项目概述

### 1.1 定位

基于价值投资和成长投资的低频量化交易系统。核心功能是**选股和择时信号生成**，不涉及自动化交易下单。使用者通过提前挂单方式执行交易。

**系统定位声明：** 本系统是一台"量化价值辅助工具"。因子模型负责统计筛选——从海量标的中找出财务扎实、估值合理的候选池；对生意的定性理解（护城河、管理层、竞争格局）由人来完成。因子打分不替代人对公司的判断，所有买卖决策最终由人做出。

### 1.2 投资哲学

- **投资周期：** 核心 3 年，上下浮动 2 年（1-5 年）。现金牛消费龙头可放宽至 5-7 年；周期性恢复标的严格 2-3 年窗口
- **策略类型：** 多因子混合 + 行业轮动偏置（方案 C/D 混合）
- **股票池上限：** 三地市场候选池 ≤ 50 支；实际持仓 ≤ 15-20 支。候选池 = "我在关注的"，持仓 = "我真敢下注的"
- **排除行业：** 医药、地产（能力圈边界，非行业判断）
- **收益计量：** 统一折算为人民币（CNY），考虑汇率变动
- **因子与人的分工：** 因子负责缩小范围（扫数据、算分位、找低估、标异常）；人负责定性判断（商业模式、护城河、管理层、最终决策）。因子是帮手，不是判官

### 1.3 目标市场

| 市场 | 数据源 |
|------|--------|
| A股（沪深） | akshare |
| 美股 | yfinance |
| 港股 | akshare + yfinance 互补 |

### 1.4 技术栈

- **语言：** Python 全栈
- **界面：** Web 应用，本地部署（Gradio）
- **存储：** SQLite（WAL 模式）+ Parquet 文件
- **不引入：** PostgreSQL、Redis、Celery、Docker（第一版）

### 1.5 运行频率

| 任务 | 频率 | 触发方式 |
|------|------|---------|
| Data Pipeline（行情） | 每日 | 自动/手动 |
| 财务数据更新 | 每季度 | 手动/财报后 |
| Factor Engine | 每日 | 自动 |
| Strategy Engine | 每日 | 自动 |
| AI 调研（漏斗能力圈层） | 按需/财报季 | 手动触发 |
| AI 确认（BUY 信号） | 按需 | 首次出信号时触发 |

### 1.6 v1 范围裁切

为避免 v1 永远做不完，明确首版交付范围和后续版本规划：

**v1 交付（最小可用）：**
- Data Pipeline：三地行情 + 财务 + 汇率 + 退市列表（akshare/yfinance）
- Factor Engine：价值/成长/质量因子 + YAML 配置 + 跨市场分层标准化
- Strategy Engine：五层漏斗（含能力圈白名单 + AI 问题生成器）、打分、PE 分位择价 + DCF 交叉验证、信号生成
- Core Engine：回测（PIT 安全 + 样本内外切分 + 分市场基准 + 低 N 警告 + regime 标注）、因子归因
- Portfolio State：手动持仓录入 + HOLD/SELL 信号联动
- 组合构建：简单规则（确信度上限 + 市场集中度上限 + 现金预留）
- Web Dashboard：四个 Tab（Gradio）
- 动量因子仅做反向校验信号，不参与综合得分
- 不包含：APScheduler 定时、行业归因、择时归因、货币归因、尾部风险压力测试

**v2 规划：**
- 定时调度（APScheduler）
- 完整业绩归因（行业/择时/货币）
- 尾部风险压力测试
- 移动端通知（Webhook/企业微信）

## 2. 系统架构

### 2.1 架构方案：模块化微内核

系统拆为六大独立模块，通过明确的数据契约通信：

```
Web Shell (Gradio)
     │
     ├── Data Pipeline     ← 多源采集、清洗、标准化、PIT 快照、本地存储
     ├── Factor Engine     ← 因子定义(YAML)、计算、跨市场分层标准化
     ├── Strategy Engine   ← 五层漏斗、多因子打分、行业轮动偏置、PE 分位择价+DCF、AI 问题生成
     ├── Core Engine       ← 回测（PIT 安全+样本内外+基准）、风险、因子归因
     └── Portfolio State   ← 手动持仓录入、持仓与信号联动、thesis 检查清单
```

### 2.2 解耦原则

- 每个模块通过明确的 Python 接口对外暴露，内部实现可独立修改
- 因子定义以 YAML 配置化，新增因子只需 YAML + 一个计算函数
- Strategy Engine 不直接读取原始数据，只接收 Factor Engine 输出的因子矩阵
- Web 壳层不包含业务逻辑，只做展示和用户交互
- 漏斗筛选、因子计算、策略打分三者独立迭代，互不影响
- Portfolio State 是独立模块，Strategy Engine 消费其数据但不对其写入

### 2.3 数据流

```
定时/手动触发
    │
    ▼
Data Pipeline ──→ 更新本地数据库（增量拉取 + 质量控制校验）
    │               写入 raw/ （不可变，日期分区）→ PIT 切片服务
    ▼
Factor Engine ──→ 对股票池 + PIT 切片日期 → 因子计算 → 市场内分层标准化
    │               动量因子独立计算（仅反向校验，不进入综合得分）
    │               扣除退市样本
    ▼
Strategy Engine ──→ 读取因子 + 持仓状态 + 行业数据
    │               → 漏斗（基础过滤→能力圈→财务标签→AI问题→估值）
    │               → 打分（因子线性加权，市场内分位秩排名）
    │               → 行业轮动偏置（不改分，仅展示配置建议）
    │               → PE 分位择价 + DCF 交叉验证
    │               → 信号输出
    ▼
Core Engine ──→ PIT 安全回测（样本内/外切分，含分市场基准对比）
    │               → 因子归因
    ▼
Web Shell ──→ Dashboard 展示信号、排名、行业分布、风险指标
```

## 3. Data Pipeline — 数据管线

### 3.1 采集数据类型

| 数据类型 | 具体内容 | 更新频率 |
|----------|---------|---------|
| 日线行情 | OHLCV（开高低收量） | 每日 |
| 财务数据 | 营收、利润、ROE、负债率、自由现金流等 | 每季度 |
| 估值数据 | PE/PB/PS/股息率/EV-EBITDA | 每日 |
| 行业分类 | 申万/GICS 行业归属 | 月度 |
| 宏观指标 | 利率、CPI、PMI、汇率 | 月度/周度 |
| 指数数据 | 沪深300/标普500/恒生 行情与估值 | 每日 |
| 汇率数据 | USD/CNY、HKD/CNY 日线 | 每日 |
| 退市列表 | 各市场历史退市股票及退市日期 | 月度 |
| 基准指数 | 沪深300价值 / 标普500价值 / 恒生综合 | 每日 |

### 3.2 本地存储

```
data/
├── raw/                       # 原始数据（不可变，日期分区）
│   ├── market/
│   │   └── YYYY-MM-DD/        # 按拉取日期分区
│   │       ├── a_share.parquet
│   │       ├── us.parquet
│   │       └── hk.parquet
│   ├── fundamental/
│   │   └── YYYY-MM-DD/        # 按财报披露日期分区（PIT 关键）
│   │       ├── a_share.parquet
│   │       ├── us.parquet
│   │       └── hk.parquet
│   └── macro/
│       └── YYYY-MM-DD/
│           ├── rates.parquet
│           └── fx.parquet
├── processed/                 # 清洗后数据
│   ├── factors/               # 计算好的因子值（按计算日期分区）
│   └── signals/               # 历史信号记录
├── metadata/                  # SQLite（WAL 模式）
│   ├── universe.db            # 股票池、行业映射、上市/退市日期、能力圈白名单
│   └── portfolio.db           # 持仓记录（见 §4.8）
└── pit/                       # PIT 快照索引（SQLite）
    └── pit_index.db           # 任意日期的可用数据版本映射
```

### 3.3 数据处理原则

- **后复权为默认存储格式：** 后复权价格不会因未来分红/拆股而修改历史价格，回测安全。Dashboard 展示时再换算前复权价格供人阅读。
- **增量更新：** 每次只拉最新数据，不做全量覆盖
- **货币单位：** 本地货币（CNY/USD/HKD），回测时按汇率折算
- **交易日对齐：** 三地市场独立维护交易日历
- **汇率折算公式（含分红）：**
  - `实际收益(CNY) = (1 + 本币总收益含分红) × (1 + 汇率变动) - 1`
  - 汇率变动 = `CNY/FX_end / CNY/FX_start - 1`（外币升值时为正，对 CNY 收益有增益）
  - 本币总收益含分红再投（与回测口径一致）
- **跨市场每日运行时序对齐：**
  - 每日触发时间：北京时间 T 日 22:00（美股 T-1 已收盘，A股/港股 T 日已收盘）
  - A股/港股使用 T 日收盘数据
  - 美股使用 T-1 日收盘数据（T 日美股尚未开盘）
  - 汇率使用 T 日（北京时间）收盘价
  - 此举确保一次 run 内所有数据的时间戳语义一致

### 3.4 点时间（PIT）数据契约

**这是回测可信度的核心约束。** 回测必须在任意历史日期 T 只能看到 T 日实际已公开的数据，不得混入未来信息。

**PIT 数据切片规则：**

| 数据类型 | PIT 约束 |
|----------|---------|
| 行情（OHLCV） | 使用后复权价。因子计算时取 T 日当天的后复权快照（通过 raw 日期分区 + 复权因子回溯得到） |
| 财务数据 | 按**财报披露日期**（非报告期）建立 PIT 视图。T 日只能看到 ≤ T 的所有已披露财报。需处理：业绩预告→业绩快报→正式报告的顺序修正 |
| PE/PB 分位 | 使用 PIT 财务数据 + PIT 价格计算。计算 lookback=5 年分位时，用 T 日可用的 5 年数据，不含 T 之后的任何数据点 |
| 行业分类 | 按历史行业归属快照（T 日时该公司属于哪个行业），不用最新分类回溯覆盖 |
| 股票池 | T 日已上市且未退市的股票（含退市前的最后交易日数据） |

**PIT 实现方式：**

- 每次拉取数据按 `YYYY-MM-DD` 分区存入 `raw/`，永久不可变
- `pit_index.db` 维护"任意日期 → 可用数据版本"的映射，回测时通过此索引取数
- Factor Engine 接收 `as_of_date` 参数，内部通过 PIT 索引获取该日期的数据版本
- 回测引擎逐日推进时，每步只传入当日 PIT 数据切片

### 3.5 幸存者偏差处理（降级为非重点）

**定位调整（v1.4）：** 退市/幸存者偏差处理不再是系统重点。理由：(1) 探针实测确认港股、美股退市列表在免费源中不可得，三地完整修正本就做不到；(2) 本系统的选股域是大市值优质股（漏斗第 1 层市值/流动性门槛已过滤掉高风险小盘股），3 年持有期内退市概率极低；(3) 完整退市修正的工程成本高、收益低。因此采用轻量方案。

**轻量处理方案：**

- **A 股：** 用 akshare 退市列表尽力修正——回测时退市前正常纳入股票池，退市日以最后交易价结算、剩余仓位记为现金。这是免费源中唯一覆盖较好的市场，修正成本低。
- **港美股：** 不做退市修正。回测报告中标注"港美股未修正幸存者偏差，收益可能被轻微高估"，将此不确定性纳入结果解读。如未来引入付费源（CRSP/Compustat）可再补全。
- 漏斗筛选时，T 日只能看到 T 日仍在市的股票（这一 PIT 约束仍保留）。
- 不再强制要求"区分归零 vs 被收购获利"等精细化退市原因处理——超出 v1 投入产出比。

### 3.6 数据质量校验与兜底

免费数据源（akshare/yfinance）不可靠，必须有防御层：

**拉取失败处理：**

| 场景 | 策略 |
|------|------|
| akshare 单次超时/报错 | 重试 3 次，指数退避（1s/3s/9s） |
| akshare 连续 3 天失败 | 标记该市场数据状态为 `STALE`，Dashboard 显示警告，不阻塞其他市场 |
| yfinance 单次超时/报错 | 重试 3 次 |
| 港股数据：akshare 和 yfinance 都拉取 | akshare 为主源，yfinance 为备源；备源数据用于交叉校验和补缺 |

**数据完整性校验（每次拉取后执行）：**

- 交易日计数：对比该市场官方交易日历，检测缺失交易日
- 异常 OHLC：Low > High、Close 为 0、单日涨跌幅超过阈值（如 50%）→ 标记异常
- 财务字段缺失率：单只股票缺失关键字段 > 30% → 写入 `data_quality_log`，该票在当次因子计算中降权处理
- 校验结果写入 `metadata/data_quality.db`，Dashboard Tab 1 可查看最近质量问题

**缺口补拉：**

- `data_quality_log` 中标记的缺口，下次拉取时优先补拉
- 财务数据更新时，对已标记缺失字段的股票做定向重拉

## 4. Strategy Engine — 策略引擎（核心）

### 4.1 漏斗筛选

```
全市场（A股5000+ / 美股8000+ / 港股2500+）
        │
        ▼
  第1层：基础过滤（全自动）
  ├─ 排除行业：医药、地产（能力圈边界）
  ├─ 市值门槛：A股 > 100亿 / 美股 > $2B / 港股 > 200亿港币
  └─ 日均成交额 > 门槛（保证流动性）
        │                                     约 15,000 → 2,000
        ▼
  第1.5层：能力圈白名单（手动维护，仅实盘生效）
  ├─ 由人工维护一个"主营可理解"标记列表（元数据存储在 universe.db）
  ├─ 判断标准："我能不能用一句话讲清楚这家公司做什么"
  ├─ 同时维护"能力圈行业/赛道"清单（由人录入，如"消费品牌/云计算/新能源制造"）
  ├─ 只通过符合能力圈标记的公司；不在圈内的业务模式直接排除
  ├─ ⚠️ 能力圈白名单**不参与回测**：回测时将第 1.5 层置空，使用"全行业（除医药地产）"替代
  │   理由：能力圈是当下的个人认知，不可 PIT 化；用今天的认知过滤历史选股池等同于前视偏差
  │   能力圈只在实盘信号生成时生效
        │                                     约 2,000 → 800
        ▼
  第2层：财务健康度（标签化，不直接排除）
  ├─ ⚠️ 标签「高杠杆」：负债率 > 70%（金融股除外）
  ├─ ⚠️ 标签「盈利弱」：过去3年有2年亏损 或 FCF为负
  ├─ ⚠️ 标签「财务疑点」：审计非标/换审计师/监管处罚
  └─ 规则：≥2 标签 → 待观察池；≤1 标签 → 正常池
        │                                     约 800 → 250
        ▼
  第3层：AI 问题生成器（核心差异化层）
  ├─ v1：仅对通过第4层估值初筛的候选做 AI 调研
  ├─ v2：扩大到全部候选
  ├─ 输出：5 个该去核查的问题 + 已知风险摘要（不输出红黄绿灯）
  ├─ 例子："1. 2025年Q3应收账款增速是否显著快于营收？2. 最近大股东有无减持？..."
  ├─ 判定权完全在人手里——AI 负责提问，人负责回答和判断
  └─ 结果缓存 180 天或财报更新时刷新（详见 §4.6）
        │
        ▼
  第4层：多因子打分 + 估值筛选
  ├─ 因子打分：价值 + 成长 + 质量因子线性加权
  ├─ PE/PB 不处于历史最高 20% 分位
  ├─ 展示：因子评分雷达图 + AI 问题清单 + 估值分位 + DCF 区间
  └─ 人工最终决策，候选池 ≤ 50，持仓 ≤ 20
```

### 4.2 多因子打分

**三大类因子（动量不参与综合得分），YAML 配置化：**

| 大类 | 示例因子 | 方向 | 在综合得分中的角色 |
|------|---------|------|-------------------|
| 价值因子 | PE分位、PB分位、股息率、FCF Yield | 越低越好 / 越高越好 | **核心**：进入综合得分 |
| 成长因子 | 营收增速(3Y CAGR)、利润增速、研发占比 | 越高越好 | **核心**：进入综合得分 |
| 质量因子 | ROE、毛利率稳定性、现金流质量、杠杆率 | 越高越好 | **核心**：进入综合得分 |
| 动量因子 | 12-1月动量、相对大盘强度 | 越高越好 | **仅反向校验**：不进入综合得分；当动量方向与估值择价方向一致时增加置信度，方向相反时触发审视提醒。权重写死为 0 |

**动量因子的定位：** 动量是 6-12 个月中短期效应，与 3 年价值持有期存在方法学张力。动量因子保留数据计算管道（方便回测时对比"有动量"和"无动量"的绩效差异），但**在综合得分中权重永久为 0**，不作为选股输入。动量数据仅用于：当某股票 PE 分位处于低估区间但动量负向 → 系统提醒"估值便宜但近期弱势"，由人评估是否为更好的入场时机。

**跨市场标准化规则（替代纯 Z-score）：**

跨市场直接 Z-score 混算会导致估值中枢不同的市场系统性占优。标准化必须分层执行：

```
第0层（必选）：因子值截尾
  → MAD 法去极值（median ± 5×MAD），截尾值置为边界值

第1层（必选）：市场内 Z-score 标准化
  → 在 A股 / 美股 / 港股 各自内部做 Z-score
  → 消除跨市场估值中枢差异

第2层（必选）：行业内 Z-score 标准化（市场-行业内）
  → 在同一市场、同一行业内做 Z-score
  → 消除跨行业财务结构差异（如科技 vs 消费的杠杆率不可比）

第3层（可选）：市值中性化
  → 对同一市场-行业内按市值分位回归取残差
  → 默认关闭，仅在发现市值因子过度支配时开启
```

**跨市场排名方式：**

- 综合得分不跨市场直接比较原始 Z 值，而是比较**市场内分位秩（percentile rank）**
- 信号 "得分前 20%" 含义：在该股票所属市场内排名前 20%
- 跨市场排序用分位秩而非绝对值，避免某市场系统性占据 Top N

### 4.3 行业轮动（降级为信息展示，不改分）

- **方式：** 行业配置参考建议，不对个股评分施加偏置
- **逻辑：** 根据宏观经济阶段（PMI/利率曲线/CPI）判断当前有利行业
- **Dashboard 展示：** 在 Tab 3（行业热力图）中显示"当前经济阶段 → 历史上该阶段哪些行业表现偏强/偏弱"，作为配置参考
- **不参与信号生成：** 行业参考信息不修改个股的综合得分和信号判定
- **理由：** 3 年持有期的长期逻辑不应被宏观短周期判断扰动

### 4.4 择价（PE 分位 + DCF 交叉验证）

> 已砍掉 200MA 趋势择时。3 年持有期内，短期价格趋势是噪声而非信号。

**估值择价（PE 分位）：**

```
PE分位 ≤ 30%    → 🟢 绿灯（估值相对自身历史处于偏低区间）
PE分位 30-50%   → 🟡 黄灯（估值处于中枢，需更严格的价格要求）
PE分位 > 50%    → 🔴 红灯（估值偏高，等待或要求更高安全边际）
```

**DCF 交叉验证（对进入第 4 层的候选股）：**

- 对候选池股票跑简化 DCF 模型（自由现金流折现），输出内在价值区间
- DCF 参数默认保守：折现率 10-12%、终值增长率 ≤ 3%、使用过去 5 年平均 FCF
- **DCF 行业适用性限制：** 金融股（银行/保险/券商/多元化金融）没有传统意义的自由现金流——其负债即为经营性资金，FCF DCF 对其无意义。金融股跳过 DCF，仅用 PE/PB 分位 + ROE 质量判断作为估值参考。DCF 计算函数需在入口处识别 GICS 行业代码（40 金融），对金融股返回 `N/A` 并标记原因
- 交叉验证逻辑：
  - PE 分位 ≤ 30% **且**（DCF 不适用 或 当前价低于 DCF 区间下限）→ **高确信**安全边际
  - PE 分位 ≤ 30% 但当前价高于 DCF 区间 → 标记为"统计便宜但 DCF 不支持"，需人工审视
  - PE 分位 > 50% 但 DCF 区间上限高于当前价 → 可能为成长被低估，标记供审视
- 周期股（大宗商品/航运/化工）的 5 年平均 FCF 在周期顶部会高估内在价值——Dashboard 对周期行业持仓额外标注"周期顶部风险"

**PE 计算口径约定：** PE = Price / EPS。后复权价格是累积复权后的数值（可远高于现价），而 EPS 是单期值。如直接用后复权价 / EPS，PE 会被系统性放大失真。**PE/PB 等估值因子的正确计算方式：**
- 方案 A（推荐）：**总市值法** — `PE = 总市值 / 归母净利润`。总市值与价格同向后复权倍率，分子分母口径一致，天然无误
- 方案 B：价格和 EPS **同向后复权** — 即 EPS 也按复权因子向后累积调整
- v1 采用方案 A（总市值法）；若数据源不提供总市值，则用方案 B 并在计算函数中做同口径复权变换

**安全边际说明：** PE 分位衡量的是"市场此刻对公司定价相对自身历史的水平"（相对安全边际）；DCF 衡量的是"价格相对内在价值的水平"（绝对安全边际）。两者交叉验证优于单一指标。最终安全边际判断由人做出。

### 4.5 信号类型

| 信号 | 含义 | 触发条件 |
|------|------|---------|
| BUY | 当前可买入 | 未持仓 + 综合得分（市场内）前 20% + PE 分位择价 OK + DCF 交叉验证不矛盾 |
| WATCH | 加入观察 | 得分前 20% 但择价/DCF 不满足，或缺少 AI 调研 |
| HOLD | 继续持有 | 已在持仓中，基本面未恶化 |
| 🚩 REVIEW | 需审视持仓 | 已有持仓，且（得分跌入后 20% **或** AI 问题生成器发现新疑点 **或** 基本面指标恶化）。触发后由人重审 thesis，不自动建议卖出 |
| 🟥 SELL | 建议卖出 | 由人手动确认 thesis 已破（竞争格局恶化/盈利衰退非周期/管理层出事/原买入理由不再成立）后标记 |

**卖出逻辑设计原则：**

- 机器负责提醒（🚩 REVIEW），人负责判决（🟥 SELL）
- 得分百分位穿越不单独触发卖出——那可能只是在交易噪声
- 卖出只有三个理由：thesis 坏了、估值极端高估、有显著更好的替代机会
- Dashboard 对每只持仓展示"你当初为什么买"（thesis 备注），与当前状态对照——这是人的判断工具，不是机器的卖出指令

### 4.6 AI 问题生成器

**定位：** AI 的职责不是替人做判断，而是帮人**问对的问题**——发现需要去查的疑点和风险点。

**成本控制 & 缓存策略：**

| 维度 | v1 策略 |
|------|--------|
| 调研范围 | 仅覆盖通过第4层估值初筛的候选（约 80 家） |
| 缓存 TTL | 180 天（覆盖一个完整财报季周期） |
| 过期条件 | 缓存 > 180 天 **或** 该公司有新财报披露 **或** 存在重大事件标记 |
| 调用限速 | 每秒 ≤ 1 次，每次 run 上限 50 次（防账单失控） |
| 降级路径 | AI 调用失败时标记"调研未完成"，不阻塞漏斗流程 |
| 密钥管理 | API key 通过环境变量 `WEBSEARCH_API_KEY` 注入，不写入配置文件或代码 |

**输出格式（替代红黄绿灯）：**

每家公司输出：
```
## [公司名] — AI 问题清单

由 AI 生成、需人工核查的 5 个问题：

1. [具体问题] — 为什么值得查：[一句话理由]
2. [具体问题] — 为什么值得查：[一句话理由]
3. [具体问题] — 为什么值得查：[一句话理由]
4. [具体问题] — 为什么值得查：[一句话理由]
5. [具体问题] — 为什么值得查：[一句话理由]

已知风险摘要（来自公开信息）：
- [风险点]
- [风险点]
```

**问题覆盖维度：**
- 财报中需重点核查的科目（应收账款异常、关联交易、商誉减值风险等）
- 商业模式可持续性疑点（客户集中度、技术替代风险）
- 管理层信号（近期减持、高管离职、股权质押）
- 竞争格局变化（新进入者、市场份额趋势）
- 未来轨迹（5 年后这家公司是否仍在牌桌上？什么可能让它掉队？）

### 4.7 组合构建

v1 采用简单规则，不做数学优化（Markowitz 不做、协方差矩阵不做）。

**仓位规则：**

| 约束 | 值 | 说明 |
|------|-----|------|
| 单票上限（高确信） | ≤ 15% | 人工标记为"高度确信"的持仓 |
| 单票上限（中确信） | ≤ 8% | 人工标记为"中度确信"的持仓 |
| 单票上限（低确信） | ≤ 3% | 试探性仓位 |
| 单一市场上限 | ≤ 60% | 避免系统性暴露于单一市场监管/汇率风险 |
| 单一行业上限 | ≤ 40% | 避免行业集中风险 |
| 现金预留 | ≥ 10% | 应对黑天鹅和新的入场机会 |

**确信度标记：** 由人工在 Dashboard Tab 2 的持仓管理面板中为每笔持仓标记高/中/低确信度。信号生成的仓位建议基于此标记。

**组合占用率实时展示：** Dashboard Tab 2 持仓管理面板顶部增加一个"组合占用率"状态条，实时显示：各确信度段的已用比例、单一市场占用比例、单一行业占用比例、现金比例。防止上限形同虚设。

**相关性约束（v2）：** v1 不做相关性矩阵约束；v2 加入持仓间相关性警告（当组合中持有高度相关的多只股票时提醒）。

### 4.8 Portfolio State — 持仓状态模块

**数据存储：** `metadata/portfolio.db` (SQLite)

```sql

holdings(code, market, shares, avg_cost, cost_currency,
         buy_date, thesis_notes, conviction,     -- thesis_notes: "当初为什么买"

-- 交易记录表
```

**对外接口：**

## 5. Factor Engine — 因子引擎

### 5.1 因子定义（YAML 配置化）

```yaml
# config/factors/value.yaml 示例
factors:
  pe_percentile:
    name: "PE历史分位"
    category: value
    direction: reverse
    params:
      lookback: 5
      trim_outliers: 0.01
      market_specific: true    # 是否做市场内标准化
      sector_neutral: true     # 是否做行业内标准化
    weight: 0.15               # 在多因子模型中的权重
    in_composite: true         # 是否进入综合得分

  fcf_yield:
    name: "自由现金流收益率"
    category: value
    direction: forward
    params:
      fcf_lookback: 5          # 5年平均FCF
      market_specific: true
      sector_neutral: true
    weight: 0.10
    in_composite: true

# config/factors/momentum.yaml
factors:
  momentum_12m1m:
    name: "12-1月动量"
    category: momentum
    direction: forward
    params:
      skip_recent: 1
      market_specific: true
      sector_neutral: false
    weight: 0.00               # 永久为0：不进入综合得分
    in_composite: false        # 仅作为估值择价的反向校验信号
```

### 5.2 计算流程

```
输入: universe + factor_config + as_of_date(PIT)
  → PIT 数据加载（通过 pit_index 获取 as_of_date 的数据版本）
  → 扣除非 as_of_date 时点的退市股票
  → 逐因子计算（每个因子独立计算函数）
  → 分层标准化（截尾 → 市场内 Z → 行业内 Z → 可选市值中性）
  → 输出: factor_matrix (DataFrame: codes × factors)
  附：各因子的市场内分位秩（用于跨市场排序）
```

### 5.3 解耦约定

- Factor Engine 不感知股票池（universe 从外部传入，含能力圈过滤后的结果）
- Factor Engine 通过 `as_of_date` 接收 PIT 时间点，内部不做数据时点判断
- 因子定义与计算分离（YAML + 独立 compute 函数）
- 标准化逻辑由 Factor Engine 内部执行，策略层只收标准化后的因子矩阵 + 分位秩
- 新增因子：加一个 YAML 条目 + 一个 compute 函数，不碰上游
- 退市股票由 Data Pipeline 维护列表，Factor Engine 在执行时从 universe 中扣除
- **因子有效性分市场独立验证：** 每个因子先在 A 股/美股/港股各自独立做有效性测试（IC/ICIR/分层回测），通过市场内检验后再谈跨市场比较（v1 实现 IC 检验，v2 加入分市场分层回测）

## 6. Core Engine — 核心引擎

### 6.1 回测

轻量自研回测框架，不引入 backtrader。

**防前视偏差（PIT 安全）：**

- 回测逐日推进，每步只传入当日 PIT 数据切片
- 财务数据按披露日期版本取用，不使用"最新一期"回溯
- 价格使用后复权历史快照（不受未来分红拆股影响）
- PE 分位在回测的每一天只用截至当日的历史数据计算
- 退市股票（降级处理，见 §3.5）：仅 A 股尽力修正（退市前正常参与，退市日以最后交易价结算）；港美股不修正，回测报告标注

**防过拟合（样本内外切分）：**

- **样本内（训练期）：** 最近 10 年的前 7 年，用于因子权重调参
- **样本外（验证期）：** 最近 10 年的后 3 年，不用于调参，仅用于最终评估
- **Walk-forward（可选，v2）：** 滚动窗口验证
- **约束：** Tab 4 的"对比回测"功能必须展示样本内/样本外的分项指标，防止只看样本内自欺
- **可调参数限制：** v1 可调参数仅为各因子权重（4-8 个），不开放因子内部参数调优（如 lookback 窗口），降低过拟合自由度

**低独立样本警告（重要）：**

- 10 年回测窗口 × 3 年持有周期 = 样本内仅 ~2-3 个独立持仓周期、样本外 ~1 个
- 此样本量下年化收益/夏普比率的统计可靠性有限
- **强制约束：** 回测结果页顶部必须显示以下警告文本（不可关闭）：
  > ⚠️ 回测期仅含 3-4 个独立持仓周期，统计指标可靠性有限。以下数字是参考，不是概率保证。不要据此过度优化因子权重。

**Regime 标注：**

- 回测收益曲线上标注牛/熊/震荡市 regime 分段
- 各 regime 下的独立绩效指标（年化收益、最大回撤），帮助理解策略对不同市场环境的适应性

**基准对比：**

| 市场 | 基准指数 | 降级方案 |
|------|---------|---------|
| A股 | 沪深300价值指数 | 若 not available → 沪深300 |
| 美股 | 标普500价值指数 | 若 not available → 标普500 |
| 港股 | 恒生综合指数 | 若 not available → 恒生指数 |

回测输出必须包含超额收益（vs 基准）和跟踪误差，否则绝对收益数字无参照意义。

**回测输入与输出：**

- 输入：策略配置 + 时间范围 + 历史股票池（含退市列表）
  - 能力圈白名单**不参与回测**：回测时第 1.5 层（能力圈过滤）置空，使用"全行业（除医药地产）"作为选股域。理由：能力圈是当下的个人认知，不可 PIT 化；用今天的能力圈过滤历史选股等同于前视偏差
- 输出：分市场累计收益曲线、超额收益曲线（vs 基准）、年化收益（样本内外分列）、夏普比率（附低 N 警告）、最大回撤（含回撤期长度）、各 regime 绩效拆解

### 6.2 风险指标

- VaR (95%)、CVaR
- 最大回撤及回撤持续时间
- 行业集中度、单票仓位上限
- 汇率风险敞口（各货币资产占比）

### 6.3 业绩归因（v1 范围）

v1 仅实现因子归因；行业归因、择时归因、货币归因、尾部风险压力测试延至 v2。

- **因子归因（v1）：** 各因子对收益的贡献度（Brinson 归因简化版）
- 行业归因（v2）：行业配置 vs 基准的差异
- 择时归因（v2）：择时决策的收益贡献
- 货币归因（v2）：汇率变动对总收益的影响
- 尾部风险压力测试（v2）：相关性飙升/汇率危机/A 股政策冲击/退市潮等极端场景

  - 标记卖出按钮（选择卖出原因：thesis坏了/高估/有更好的/其他 → 录入卖出价）
  - 🚩 审视中的持仓高亮显示，附带触发审视的原因

### Tab 3：行业热力图
- 当前经济阶段判定（基于 PMI/利率曲线/CPI）
- 该阶段历史上各行业表现参考（仅展示，不修改信号）
- 各行业当前得分分布 / 候选池数量 / 持仓数量

### Tab 4：回测 & 配置
- 回测面板：选择策略配置 → 选时间范围 → 运行回测
  - ⚠️ 低 N 警告（不可关闭）
  - 收益曲线（样本内/样本外分色标注，regime 标注）
  - 超额收益曲线 vs 基准
  - 分项指标表格（分市场、分样本内/外、分 regime）
- 因子配置管理：YAML 编辑 → 重新计算 → 对比回测（强制展示样本内/外）

### SQLite 并发安全

- 所有 SQLite 连接统一开启 **WAL 模式**（`PRAGMA journal_mode=WAL`）
- 读操作可并发，写操作在应用层串行化（单线程写锁）
- Gradio/调度/手动触发共享同一个数据库连接池

## 10. 测试策略

### 10.1 数据质量测试（`tests/test_quality/`）

- OHLC 异常值断言：Low > High、Close ≤ 0、单日涨跌 > 50% 自动检出
- 交易日缺口检测：对比官方交易日历，缺失日 > 阈值即告警
- 财务字段缺失率：全市场字段缺失率监控，异常波动告警

### 10.2 因子计算正确性测试（`tests/test_factors/`）

- 选取 5 支已知股票、固定日期范围，手工计算 PE 分位/ROE/DCF
- 测试用例 = 手算结果 ± 1e-6
- 覆盖去极值、市场内标准化、行业内标准化后的输出数值

### 10.3 PIT 无前视测试（`tests/test_pit/`）

- 选取一个已知财报披露日 D，测试 D-1 日的 PIT 切片不包含该财报数据
- 测试后复权价在分红除权日前后的一致性
- 测试 PE 分位计算在回测的每一天只使用截至当日的历史数据

### 10.4 回测边界测试（`tests/test_backtest/`）

- 退市股票在退市日正确结算，不再出现于后续交易日
- 样本内/外切分边界无数据泄露
- 空股票池、单股票池的边界行为

## 11. 设计决策记录

| 决策点 | 选择 | 理由 |
|--------|------|------|
| 系统定位 | 量化价值辅助工具 | 因子缩小范围，人做定性判断和最终决策 |
| 架构 | 模块化微内核（6 模块） | 策略频繁微调，需要模块独立迭代 |
| 漏斗方式 | 五层漏斗（含能力圈白名单 + AI 问题生成器） | 先能力圈、再财务、再估值，AI 帮人提问而非替人判断 |
| 行业轮动 | 降级为信息展示，不修改评分 | 3 年持有期的长期逻辑不应被宏观短周期扰动 |
| 动量因子 | 权重写死为 0，仅反向校验 | 6-12 月动量与 3 年持有期存在方法学张力 |
| 择时 | 砍掉趋势择时，保留 PE 分位择价 + DCF | 200MA 与长期投资逻辑矛盾；PE 分位+DCF 交叉验证提供更可靠的安全边际参考 |
| 卖出逻辑 | 🚩审视（机器提醒）+ 🟥卖出（人确认） | 机器人不替人做卖出决策 |
| AI 角色 | 问题生成器，不输出红黄绿灯 | 避免 AI 幻觉制造虚假确信；判定权留给人 |
| 组合构建 | 简单规则（确信度上限 + 集中度上限 + 现金预留） | v1 无需 Markowitz 优化 |
| 存储 | SQLite（WAL）+ Parquet | 单人零运维 |
| 价格格式 | 后复权（回测默认） | 不会被未来分红/拆股修改历史价格 |
| 回测 | 自研 + PIT 安全 + 分市场基准 + 低 N 警告 | 低频逻辑简单，框架无法实现 PIT 精细控制 |
| 收益计量 | 统一 CNY（含分红+汇率） | 跨市场真实比较 |
| 跨市场标准化 | 市场内 → 行业内分层 Z-score | 避免估值中枢差异导致某市场系统性占优 |
| 因子有效性 | 分市场独立 IC/ICIR 检验 | 不同市场的因子行为可能有结构性差异 |
| 前端 | Gradio | Python 一体，无需写 JS |
| AI 调研范围 | v1 仅覆盖估值层候选（~80家） | 控制成本和开发量 |
| 归因范围 | v1 仅因子归因 | 先跑通核心链路 |
| 压力测试 | v2 | v1 先完成核心回测链路 |
| 能力圈在回测中 | 不参与 | 能力圈是当下认知，不可 PIT 化，回测时置空 |
| DCF 对金融股 | 跳过，仅用 PE/PB + ROE | FCF DCF 对金融股无意义 |
| PE 计算口径 | 总市值法（总市值/归母净利润） | 后复权价直接除 EPS 会系统性放大 PE，总市值法口径天然一致 |
| 退市处理 | 降级为非重点：仅 A 股修正，港美股标注 | 免费源不可得 + 大市值选股域退市概率极低 + 投入产出比低（见 §3.5） |

## 12. 开工前数据探针清单

以下验证**必须在写第一行策略代码前完成**——验证的是数据源的底层可用性，而非代码缺陷。若某项失败，对应设计需在开工前调整。

> **v1.4 优先级标注：** 探针 2、3（港股/美股退市列表）已降级为**参考性低优先级**探针——退市处理整体降为非重点（§3.5），这两个探针仅用于确认"免费源不可得"这一事实并触发标注路径，其结果不阻塞策略开发。探针 1、4、5 仍是硬门槛。

| # | 探针项 | 验证方法 | 通过标准 | 失败时的降级方案 |
|---|--------|---------|---------|----------------|
| 1 | **A 股历史财报披露日期覆盖率** | 用 akshare 拉取沪深300成分股最近 10 年财报的 `announcement_date`（或等同字段），统计非空率 | 覆盖率 ≥ 80% | 用"报告期 + 固定滞后"近似：年报滞后 4 个月、中报滞后 2 个月、季报滞后 1 个月；在 §3.4 标注 PIT 精度降级为"近似披露日" |
| 2 | **港股退市列表完整性** | 用 akshare 拉取港股历史退市列表，抽查过去 5 年已知重大退市案例（如私有化、收购）是否在列表中 | 抽查 5 例中 ≥ 4 例命中 | 在回测报告中标注"港股退市数据覆盖率偏低，回测收益可能偏高"；人工补录重大案例 |
| 3 | **美股退市列表可行性** | yfinance 不提供退市列表；尝试其他免费源（WRDS 学术账号？NASDAQ 退市页面？）或确认无法获取 | 至少确认是否有替代源 | 若不可取：回测仅对 A 股做退市修正，美/港股回测标注"未修正幸存者偏差，收益可能被高估" |
| 4 | **价值风格基准指数覆盖率** | 用 akshare/yfinance 拉取沪深300价值指数最近 10 年日线，确认序列完整 | 缺失日 ≤ 5% | 降级为沪深300（已在 §6.1 基准表中标注降级路径） |
| 5 | **财务数据字段完整性** | 对三地各抽 20 支大/中/小盘股，拉取最近 5 年 ROE/FCF/营收/净利润/总市值字段，统计字段级缺失率 | 单市场单字段缺失率 ≤ 15% | 缺失字段从因子体系中移除该股该期的计算点，在 `data_quality_log` 中记录 |

探针脚本放在 `src/data_pipeline/probes/` 下，每个探针独立可运行，输出 `passed/failed/warning + 统计摘要`。全部通过后再动工写策略代码。探针结果应纳入 `data_quality.db` 并在 Dashboard Tab 1 中可查看（作为"数据源健康度"面板）。
